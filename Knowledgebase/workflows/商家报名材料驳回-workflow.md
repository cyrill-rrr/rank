# 商家报名材料驳回

## 适用场景

商家提交的报名材料不满足要求，需要由审核人员或系统将材料驳回，并允许商家根据规则重新提交。

## 需要输入

| 输入 | 用途 |
| --- | --- |
| 报名记录 | 确认材料属于哪次报名 |
| 材料记录 | 定位要驳回的材料版本 |
| 驳回原因 | 用于通知商家和后续追溯 |
| 操作人 | 用于审计和责任追踪 |

## 执行步骤

| 步骤 | 动作 | 规则 |
| --- | --- | --- |
| 1 | 确认报名有效 | 报名不存在、取消或过期时，不应直接驳回材料，先回到报名域确认 |
| 2 | 确认材料可驳回 | 只有处于可审核状态的材料才能驳回；真实可驳回状态待补充 |
| 3 | 选择驳回原因 | 驳回原因来源待确认；如果必须来自枚举，不允许手写自由文本 |
| 4 | 执行驳回 | 材料状态进入 rejected 或真实等价状态 |
| 5 | 通知商家 | 通知渠道、文案和触达策略待补充 |
| 6 | 等待重新提交 | 重新提交是否生成新版本或覆盖当前版本待材料域确认 |
| 7 | 记录审计 | 记录操作人、操作时间、驳回原因和材料版本 |

## 可能结果

| 结果 | 说明 |
| --- | --- |
| 驳回成功 | 材料进入驳回状态，商家可按规则重新提交 |
| 不允许驳回 | 当前报名或材料状态不支持驳回 |
| 驳回原因缺失 | 不能执行驳回，需要先补充或选择合法原因 |
| 通知失败 | 材料状态已变更，但商家未收到通知，需要补偿或人工处理 |
