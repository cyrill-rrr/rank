# 报告未透出问题复盘

> 这是完整虚拟示例，用于演示 COE 写法。真实项目使用时应替换为经过确认的事实和来源。

## 摘要

2026-06-23，部分已完成模拟打分的商家在榜单落地页没有看到结果报告。问题表现为后台存在评分结果，但前台报告未透出。

## 影响范围

| 维度 | 内容 |
| --- | --- |
| 影响用户 | 虚拟示例：部分已报名并完成评分的商家 |
| 影响功能 | 榜单落地页、结果报告透出 |
| 影响时间 | 虚拟示例：2026-06-23 10:00 到 11:20 |
| 业务影响 | 用户无法自助查看结果，需要客服介入解释 |

## 时间线

| 时间 | 事件 |
| --- | --- |
| 10:00 | 用户反馈榜单落地页无结果 |
| 10:15 | 研发确认报名、材料、评审和打分链路均有数据 |
| 10:40 | 定位到报告域透出规则未命中 |
| 11:00 | 修复透出规则配置 |
| 11:20 | 确认受影响报告恢复展示 |

## 根因

虚拟根因：报告域透出规则遗漏了某个评分完成状态，导致报告已生成但未进入可见状态。

## 处置过程

1. 使用 [../workflows/为什么我没有上榜-workflow.md](../workflows/为什么我没有上榜-workflow.md) 从报名、材料、评审、模拟打分、报告域逐层排查。
2. 确认上游报名、材料、评审和打分均正常。
3. 在报告域发现报告生成成功但 `visible` 规则未通过。
4. 更新透出规则配置并验证前台恢复。
