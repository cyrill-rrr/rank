## 需求背景

用户完成报名、材料提交、评审和模拟打分后，需要在榜单落地页查看结果报告。过去页面只展示是否上榜，缺少对未上榜或结果不可见原因的解释。

## 目标

- 在结果页展示报告域产出的结果状态。
- 为未上榜用户提供可解释原因。
- 让客服和运营能够按照统一口径答疑。

## 影响领域

| 领域 | 影响 |
| --- | --- |
| sign | 需要判断是否存在有效报名 |
| material | 需要判断材料是否完整或被驳回 |
| review | 需要判断评审结论 |
| score | 需要判断是否完成模拟打分 |
| report | 负责结果报告生成、透出和解释 |

## 提炼出的业务规则

- 结果页展示以报告域状态为准。
- 未上榜原因应尽量映射到用户可理解的解释。
- 报告域只能消费上游事实，不应在展示层改写上游报名、材料、评审或评分结果。
- 当报告未生成或不可见时，需要给出可排查的内部原因。

## 流程变化

1. 用户进入榜单落地页。
2. 页面请求报告域结果。
3. 报告域根据报名、材料、评审、模拟打分结果返回可见状态和解释。
4. 页面展示上榜、未上榜或暂不可见说明。

## 验收标准

| 场景 | 预期 |
| --- | --- |
| 用户已上榜 | 展示榜单结果和报告 |
| 用户未上榜但链路完整 | 展示未上榜解释 |
| 材料被驳回 | 展示材料相关说明或引导 |
| 报告生成失败 | 展示暂不可见说明，并可被内部排查 |
